# brepjs-gears

Bevel gear generation for [brepjs](https://github.com/andymai/brepjs), ported from
[py_gearworks](https://github.com/GarryBGoode/py_gearworks) (Python / build123d).

Gears are generated with mathematically accurate **spherical involute (octoid)**
tooth profiles, including undercut (trochoid) calculation, tip truncation, and
optional spiral (helical) teeth.

## Install

```bash
npm install brepjs-gears brepjs occt-wasm
```

## Usage

```ts
import { OcctKernel } from 'occt-wasm';
import { registerKernel, OcctWasmAdapter, exportSTEP, unwrap } from 'brepjs';
import { BevelGear } from 'brepjs-gears';

const kernel = await OcctKernel.init();
registerKernel('occt-wasm', OcctWasmAdapter.fromKernel(kernel));

// a pair of meshing straight bevel gears on perpendicular axes
const z1 = 8, z2 = 21;
const gamma = Math.atan2(z1, z2);

const pinion = new BevelGear({
  numberOfTeeth: z1,
  module: 2,
  height: 10,
  coneAngle: 2 * gamma,
  helixAngle: Math.PI / 6, // spiral bevel; use 0 for straight teeth
});

const solid = pinion.buildSolid();
const blob = unwrap(exportSTEP(solid as never));
// write blob to a .step file
```

### Parameters (`BevelGearParams`)

Mirror of `py_gearworks.BevelGear`:

| Param | Default | Description |
|---|---|---|
| `numberOfTeeth` | — | Number of teeth |
| `coneAngle` | `PI/2` | Full pitch cone angle (radians); `2 * atan2(z1, z2)` for a 90° pair |
| `helixAngle` | `0` | Spiral angle (radians); meshing gears need opposite signs |
| `height` | `1` | Tooth length along the cone surface (not the z extent) |
| `module` | `1` | Gear module; pitch radius is `module * numberOfTeeth / 2` |
| `enableUndercut` | `true` | Compute trochoid undercut for small tooth counts |
| `tipTruncation` | `0.1` | Tip truncation coefficient for sharp tips |
| `pressureAngle` | `20°` | Pressure angle (radians) |
| `addendumCoefficient` / `dedendumCoefficient` | `1.0` / `1.2` | Tooth height coefficients |
| `profileShift`, `backlash`, `crowning`, `zAnchor` | `0` | Same semantics as py_gearworks |

Other entry points: `gear.profileAtZ(z)` samples the analytic tooth profile of a
slice (module-1 reference space), `octoid(t, params)` evaluates the raw
spherical-involute curve.

## How it differs from py_gearworks

The tooth math (octoid curves, undercut trim, tip reduction, profile assembly)
is a faithful port, validated against fixture data generated by py_gearworks
itself (see `tests/`). The B-rep construction differs in one way: instead of
fitting rational Bezier patches to profile slices (scipy optimization +
`Face.make_bezier_surface`), the analytically exact slice wires are lofted with
OCCT ThruSections, and the resulting strip shell splits a sphere/cone blank —
the same blank construction py_gearworks uses. For straight bevels the ruled
loft is geometrically identical; for spiral bevels the loft interpolates the
same sections the Bezier fit approximates.

One deliberate deviation: py_gearworks keeps the tiny self-intersection wedge
of the trochoid loop at the tooth root (its splitter tolerates the
self-intersecting boundary); this port trims the profile at the loop crossing,
which removes ~0.1% of volume below the working flank and keeps the cutting
shell well-formed.

Not (yet) ported: `meshTo` positioning, cylindrical (spur/helical) gears, ring
gears, racks, cycloid profiles.

## Tests

```bash
npm test
```

The test suite compares generated gears against reference STEP files and
intermediate math fixtures produced by py_gearworks (octoid curve samples,
profile slice samples at 1e-6, solid volume / bounding box / center of mass /
cross-section areas / boundary proximity).
